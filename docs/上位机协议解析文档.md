# BRICOS 上位机协议解析文档

本文档对应当前 BRICOS 固件的串口协议实现，给上位机提供统一的帧格式、命令字定义和 payload 解析规则。

## 1. 通道划分

- `USART3`（RT 通道）
  - 发送本体状态 `DATA_TYPE_STATE_BODY`
  - 接收运动控制命令 `DATA_TYPE_THRUSTER`
- `UART4`（NRT 通道）
  - 发送系统状态 `DATA_TYPE_STATE_SYS`
  - 发送执行机构状态 `DATA_TYPE_STATE_ACTUATOR`
  - 发送日志 `DATA_TYPE_LOG`
  - 接收系统模式、运动模式、舵机、OTA 等命令

## 2. 通用帧格式

所有通道都使用同一帧格式。

```text
| Start1 | Start2 | CmdId | Len | Payload... | Checksum | End1 | End2 |
|  0xAA  |  0xBB  |  1B   | 1B  |   Len 字节 |   1B     | 0xCC | 0xDD |
```

- `CmdId` 为 1 字节命令字
- `Len` 为 payload 长度，1 字节
- `Checksum` 为异或校验
- `TotalLen = Len + 7`

### 校验规则

`Checksum = XOR(CmdId, Len, Payload[0...Len-1])`

### 解析建议

1. 先找帧头 `0xAA 0xBB`
2. 读取 `CmdId` 和 `Len`
3. 检查 `TotalLen = Len + 7`
4. 检查帧尾 `0xCC 0xDD`
5. 计算异或校验并比对
6. 根据 `CmdId` 路由到具体解析器

## 3. 命令字定义

| CmdId | 方向 | 名称 | Payload 说明 |
| --- | --- | --- | --- |
| `0x01` | 上位机 -> MCU | `DATA_TYPE_THRUSTER` | 运动控制命令 |
| `0x02` | MCU -> 上位机 | `DATA_TYPE_STATE_BODY` | 本体状态 |
| `0x03` | MCU -> 上位机 | `DATA_TYPE_STATE_SYS` | 系统状态 |
| `0x04` | MCU -> 上位机 | `DATA_TYPE_STATE_ACTUATOR` | 执行机构状态 |
| `0x05` | MCU -> 上位机 | `DATA_TYPE_LOG` | 日志文本 |
| `0x10` | 上位机 -> MCU | `DATA_TYPE_OTA` | OTA 触发 |
| `0x11` | 上位机 -> MCU | `DATA_TYPE_SET_PID_PARAM` | PID 参数设置，当前实现仍为预留 |
| `0x12` | 上位机 -> MCU | `DATA_TYPE_SET_SYS_MODE` | 系统模式切换 |
| `0x13` | 上位机 -> MCU | `DATA_TYPE_SET_MOTION_MODE` | 运动模式切换 |
| `0x14` | 上位机 -> MCU | `DATA_TYPE_SET_SERVO` | 舵机角度设置 |

## 4. 上行数据解析

### 4.1 `DATA_TYPE_STATE_BODY` (`0x02`)

来源：`USART3`

Payload 长度：`52` 字节

字段顺序均为 `float`，按小端字节序传输。

| 偏移 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 0 | `roll` | `float` | 横滚角，单位度 |
| 4 | `pitch` | `float` | 俯仰角，单位度 |
| 8 | `yaw` | `float` | 偏航角，单位度 |
| 12 | `gyro_x` | `float` | X 轴角速度 |
| 16 | `gyro_y` | `float` | Y 轴角速度 |
| 20 | `gyro_z` | `float` | Z 轴角速度 |
| 24 | `vel_x` | `float` | X 轴速度 |
| 28 | `vel_y` | `float` | Y 轴速度 |
| 32 | `vel_z` | `float` | Z 轴速度 |
| 36 | `acc_x` | `float` | X 轴加速度 |
| 40 | `acc_y` | `float` | Y 轴加速度 |
| 44 | `acc_z` | `float` | Z 轴加速度 |
| 48 | `depth_m` | `float` | 当前深度，单位米 |

### 4.2 `DATA_TYPE_STATE_SYS` (`0x03`)

来源：`UART4`

Payload 长度：`33` 字节

字段顺序如下：

| 偏移 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 0 | `sys_mode` | `uint8_t` | 系统模式 |
| 1 | `motion_mode` | `uint8_t` | 运动模式 |
| 2 | `water_temp_c` | `float` | 外水温 |
| 6 | `cabin_temp_c` | `float` | 舱内温度 |
| 10 | `cabin_humi` | `float` | 舱内湿度 |
| 14 | `bat_voltage_v` | `float` | 电池电压 |
| 18 | `bat_current_a` | `float` | 电池电流 |
| 22 | `chip_temp` | `float` | 芯片温度 |
| 26 | `cpu_usage` | `float` | CPU 使用率 |
| 30 | `is_leak_detected` | `uint8_t` | 漏水标志，0/1 |
| 31 | `is_imu_error` | `uint8_t` | IMU 异常标志，0/1 |
| 32 | `is_voltage_error` | `uint8_t` | 低压标志，0/1 |

### 4.3 `DATA_TYPE_STATE_ACTUATOR` (`0x04`)

来源：`UART4`

Payload 长度：`3` 字节

| 偏移 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 0 | `servo_angle` | `uint8_t` | 舵机角度，0-180 |
| 1 | `light1_pwm` | `uint8_t` | 探照灯 1 亮度，0-100 |
| 2 | `light2_pwm` | `uint8_t` | 探照灯 2 亮度，0-100 |

### 4.4 `DATA_TYPE_LOG` (`0x05`)

来源：`UART4`

Payload 为日志字符串，不是二进制结构体。

- 当前内容为 UTF-8 / ASCII 文本
- 末尾带 `\r\n`
- 当前实现最大 payload 长度约为 `189` 字节
- 上位机可以直接按字符串显示，或按前缀分类处理

### 4.5 `DATA_TYPE_CMD_ACK` (`0xFF`)

来源：`UART4`

Payload 长度：`4` 字节

| 偏移 | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| 0 | `cmd_ack`   | `uint8_t` | ACK ID |
| 1 | `cmd_id`    | `uint8_t` | 对应命令 ID |
| 2 | `ack_code`  | `uint8_t` | ACK 状态码 |
| 3 | `seqnumber` | `uint8_t` | 序列号 |

## 5. 下行命令解析

### 5.1 `DATA_TYPE_THRUSTER` (`0x01`)

来源：`USART3`

Payload 长度：`17` 字节

```text
byte 0   : target_mode
byte 1-16: 4 个 float，小端序
```

`target_mode` 与 4 个 float 的语义如下：

- `0` = `MOTION_STATE_MANUAL`
  - `float0` = `surge`
  - `float1` = `sway`
  - `float2` = `heave`
  - `float3` = `yaw_cmd`
- `1` = `MOTION_STATE_STABILIZE`
  - `float0` = `surge`
  - `float1` = `sway`
  - `float2` = `target_depth`
  - `float3` = `target_yaw`
- `2` = `MOTION_STATE_AUTO`
  - 当前固件已预留，payload 仍采用同样的 4 个 float

### 5.2 `DATA_TYPE_OTA` (`0x10`)

Payload 长度：`4` 字节

固定为：

```text
DE AD BE EF
```

收到后会触发进入 bootloader。

### 5.3 `DATA_TYPE_SET_PID_PARAM` (`0x11`)

Payload 目前仍是预留项，当前固件未完成完整参数写入逻辑。

### 5.4 `DATA_TYPE_SET_SYS_MODE` (`0x12`)

Payload 长度：`1` 字节

| 值 | 说明 |
| --- | --- |
| `0` | `SYS_MODE_STANDBY` |
| `1` | `SYS_MODE_ACTIVE_DISARMED` |
| `2` | `SYS_MODE_MOTION_ARMED` |

### 5.5 `DATA_TYPE_SET_MOTION_MODE` (`0x13`)

Payload 长度：`1` 字节

| 值 | 说明 |
| --- | --- |
| `0` | `MOTION_STATE_MANUAL` |
| `1` | `MOTION_STATE_STABILIZE` |
| `2` | `MOTION_STATE_AUTO` |

### 5.6 `DATA_TYPE_SET_SERVO` (`0x14`)

Payload 长度：`1` 字节

| 字段 | 说明 |
| --- | --- |
| `servo_angle` | 舵机角度，0-180 |

## 6. 上位机实现建议

### 6.1 串口分工

- `USART3`：建议作为高实时通道，重点处理本体状态和控制命令
- `UART4`：建议作为慢速管理通道，处理系统状态、执行机构状态、日志和配置命令

### 6.2 解包策略

- 每个串口独立维护一个接收缓存
- 先按帧头同步，再按 `Len` 定长取包
- 一个串口上可能同时出现日志帧和状态帧，必须按 `CmdId` 分流

### 6.3 浮点处理

- 所有 `float` 字段都是 4 字节小端序
- 上位机解析时直接按小端浮点还原即可
- 如果上位机运行在非小端平台，需要先做字节序转换

## 7. 当前实现备注

- `USART3` 的本体状态和控制命令是强实时链路
- `UART4` 现在也已经统一到帧协议，不再使用裸文本流
- 日志虽然是文本内容，但也被封装在协议帧中，方便和二进制状态包共线传输
- 如果后续增加字段，建议同步更新：
  - `CmdId`
  - payload 长度
  - 上位机解析表
